1. 프로젝트 개요

  • 프로젝트명: AI News Platform (Sosig Shop)
  • 목표: 매일 엄선된 AI/경제 뉴스를 크롤링하여 자동 요약 후 팟캐스트로 제공
  • 기술: Python (Flask), MariaDB, Google Gemini API, Google Cloud TTS, Windows Task Scheduler
  • 인프라: Windows PC (자동 실행), Remote Server (Web/DB)
  • 개발 기간: 2025년 11월 ~ 진행 중

3. 주요 기능

📰 자동 뉴스 크롤링

Windows Task Scheduler가 매시 정각 Python 크롤러를 실행하여 최신 AI/경제 뉴스를 수집.

🤖 AI 요약 & 대본 (Gemini)

Google Gemini 2.5 Flash 모델이 '상현(선배)'과 '지민(후배)'의 티키타카 대본으로 재구성.

🎙️ 고품질 팟캐스트 (Google TTS)

Google Cloud TTS의 WaveNet 음성을 사용하여 실제 라디오 방송 같은 자연스러운 오디오 생성.

📅 캘린더 UI

날짜별, 키워드별로 뉴스를 탐색 가능.

⚙️ 관리자 대시보드

키워드 관리, 우선순위 설정, 통계 확인.

🔒 보안 게이트

3D 지구본 인터랙션 챌린지로 관리자 페이지 보안 강화.

4. 개발 스토리

Google Antigravity AI를 활용하여 전체 프로젝트를 개발했습니다.

개발 과정

  • Phase 1: 크롤러 구현 및 데이터베이스 스키마 설계
  • Phase 2: Flask 백엔드 및 REST API 개발
  • Phase 3: 반응형 웹 프론트엔드 제작
  • Phase 4: 관리자 대시보드 및 보안 강화
  • Phase 5: UI/UX 개선 (다크 모드, 애니메이션)
  • Phase 6: Google Cloud Run 마이그레이션 및 완전 자동화
  • Phase 7: Topic/Keyword 분리 시스템 및 Cloud Run 메모리 최적화 (2025.12)
  • Phase 8: Windows로컬 PC 마이그레이션 - API Rate Limit 최적화 및 네이버 뉴스 우선 추출 (2025.12)

기술적 도전

  • 로컬 실행 최적화: Windows Task Scheduler 기반 자동화로 Cloud Run의 SFTP 제약 및 네트워크 제한 문제 해결
  • IP 차단 해결: 무료 TTS 서비스의 IP 차단 문제를 Google Cloud TTS 공식 API 도입으로 해결
  • 프롬프트 엔지니어링: 단순 요약을 넘어선 '페르소나(상현/지민)' 기반의 몰입감 있는 대본 생성
  • 자동화 파이프라인: Cloud Scheduler와 연동하여 사람의 개입 없는 24/7 뉴스 생성 시스템 구축
  • Topic/Keyword 분리: 검색어(Keyword)와 표시명(Topic)을 분리하여 유연한 카테고리 관리
  • API Rate Limit 보호: Gemini API 호출 후 7초 대기 및 실패 시 30초 재시도로 429 오류 방지
  • 콘텐츠 추출 개선: 네이버 뉴스 링크 우선 사용으로 본문 추출 성공률 향상
